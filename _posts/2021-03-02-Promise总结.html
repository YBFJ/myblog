# Promise 概览

在看了三元大神的博客之后，我发现他总结的非常好三元写的关于 promise (opens new window)

promise 利用了三大技术手段来解决掉回调地狱

  • 回调函数延迟绑定
  • 返回值穿透
  • 错误冒泡

详细来说就是：

  1. 回调函数不是直接声明的，而是在通过后面的 then 方法传入的，也就是回调函数的延迟绑定
  2. 可以根据 then 中回调函数的传入值创建不同类型的 promise，然后把返回的 promise 穿透到外层使用
  3. 前面的错误可以一直向后传递，被 catch 到，也就是错误冒泡，就不用频繁检查错误
readFilePromise("1.json")
  .then((data) => {
    return readFilePromise("2.json")
  })
  .then((data) => {
    return readFilePromise("2.json")
  })
  .then((data) => {
    yyy
  })
  .catch((err) => {
    xxx
  })

# 简易版Promise

以下就是简易版本的 promise，无法实现链式调用

  1. promise 中其实传入的是一个 executor，excutor 中传入了定义在 promise 中的 resolve 函数和 reject 函数；
  2. 当 excutor 中的 resolve 函数或者 reject 函数被触发了，就会改变 promise 中定义 status 的状态，并将 value 和 reason 的值进行赋值，并传给 then 中接收的成功和失败的回调函数
function Promise(executor){
	let self = this
  self.status = "pending"
  self.value = undefined
  self.reason = undefined
  function resolve(value){
  	if(self.status === "pending"){
      self.status = "resolved"
      self.value = value
  }
  function reject(reason){
  	if(self.status === "pending"){
    	self.status = "rejected"
      self.reason = reason
    }
  }
  try{
  	executor(resolve, reject)
  }catch(e){
  	reject(e) //捕获异常就直接失败
  }
}

Promise.prototype.then = function(onFufiled, onRejected){
	let self = this
  if(self.status === "resolved"){
  	onFufiled(self.value)
  }
  if(self.status === "rejected"){
  	onRejected(self.reason)
  }
}

  let promise = new Promise(function(resolve, reject){
  	resolve(100)
  })

  promise.then(function(data){
  	console.log('data', data)
  },
    function(err){
  	console.log('err', err)
  }

)

# ES6 版本的 Promise，能够实现链式调用

区别就是将成功函数和失败函数加入了队列，从而可以进行链式调用

class promise {
  // 容纳失败后的函数们
  queue1 = []
  // 容纳成功后的函数们
  queue2 = []
  constructor(fn) {
    // 要等一会，否则queue1和queue2为空
    const resolve = (data) => {
      setTimeout(() => {
        for (let i = 0; i < this.queue1.length; i++) {
          this.queue1[i](data)
        }
      })
    }
    const reject = (reason) => {
      setTimeout(() => {
        for (let i = 0; i < this.queue2.length; i++) {
          this.queue2[i](reason)
        }
      })
    }
    fn(resolve, reject)
  }
  then(s, e) {
    this.queue1.push(s)
    this.queue2.push(e)
    // 为了下一步的链式调用
    return this
  }
}
const p1 = new Promise((resolve, reject) => {
  console.log("hi")
  if (Math.random() > 0.5) {
    resolve("大")
  } else {
    reject("小")
  }
})
p1.then(
  (data) => {
    console.log("success")
  },
  (reason) => {
    console.log("failed")
  }
)
  .then(
    () => {
      console.log("成功2")
    },
    () => {
      console.log("失败2")
    }
  )
  .then(
    () => {
      console.log("成功3")
    },

      console.log("失败3")
